# AionUI 集成测试说明

## 最新修复

已修复 AionUI 的 IPC 桥接系统初始化问题。现在 AionUI 应该能够正确加载智能体列表和技能列表。

## 如何测试

### 方法 1: 使用测试脚本（推荐）

```bash
./launch-aionui-test.sh
```

这个脚本会：
- 启动 AionUI
- 显示控制台输出
- 提供检查清单

### 方法 2: 直接命令行

```bash
"../VSCode-darwin-arm64/Code - OSS.app/Contents/MacOS/Code - OSS" --aionui
```

## 检查项目

启动后请检查以下内容：

### 1. 窗口和界面
- [ ] AionUI 窗口是否成功打开
- [ ] DevTools 是否自动打开（用于调试）
- [ ] 界面是否正常渲染

### 2. 多智能体模式
- [ ] 是否显示多智能体图标：✨ 🔄 | 🤖 OpenCode | 💎
- [ ] 是否显示"进入多智能体模式"的提示

### 3. 技能列表
- [ ] 是否显示技能列表
- [ ] 技能示例：
  - Morph PPT
  - Star Office 助手
  - 故事角色扮演
  - Beautiful Mermaid
  - moltbook
  - Cowork
  - OpenClaw 部署专家

### 4. 控制台输出（DevTools）

#### 正常输出示例：
```
[AionUi:process] initStorage +XXms
[AionUi:process] ExtensionRegistry +XXms
[AionUIWindowManager] - AionUI process system initialized successfully
[AionUIWindowManager] - bridge adapter initialized for window
build.buildStorage global
build.buildStorage system
build.buildStorage agent.chat
build.buildStorage agent.chat.message
build.buildStorage agent.config
build.buildStorage agent.env
```

#### 可以忽略的警告：
- Sentry SDK 相关警告（正常，因为没有配置 Sentry）
- Autofill.enable 相关错误（正常，DevTools 功能）

#### 需要关注的错误：
- IPC 相关错误
- Bridge 相关错误
- "office-ai-bridge-adapter" 相关错误
- 数据加载失败

## 问题排查

### 如果窗口没有打开
1. 检查是否有错误输出
2. 查看 VS Code 日志
3. 确认包构建是否成功

### 如果界面显示但没有数据
1. 打开 DevTools（应该自动打开）
2. 查看 Console 标签页
3. 查找 IPC/bridge 相关错误
4. 检查网络请求是否失败

### 如果有 IPC 错误
1. 检查 `initializeProcess()` 是否成功调用
2. 检查 `initMainAdapterWithWindow()` 是否成功调用
3. 查看主进程日志

## 技术细节

### 修复内容
- 移除了手动 IPC 处理代码
- 添加了 AionUI 进程系统初始化 (`initializeProcess()`)
- 添加了窗口桥接适配器初始化 (`initMainAdapterWithWindow()`)

### 关键文件
- `src/vs/aionui/electron-main/aionuiWindowManager.js` - 窗口管理器
- `AIONUI_IPC_FIX_REPORT.md` - 详细技术报告
